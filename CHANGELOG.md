# 更新日志

本项目遵循[语义化版本](https://semver.org/lang/zh-CN/)，版本号单一来源为 [`src/version.h`](src/version.h)。
发布 tag（如 `v1.0.9`）必须与该文件一致，否则发布流水线会失败。

## [1.0.14] - 2026-07-20

### 新增

- **在桌面上画「最大化 / 还原」可以把刚最小化的窗口拿回来**。用手势最小化一个窗口后桌面就空了，
  此时在桌面上画 `cmd:maximize` / `cmd:restore` / `cmd:toggle_maximize`（默认配置里是 `9` ↗），
  会恢复最近一次被 hua 最小化的那个窗口——这几个手势原本以桌面为目标时是空操作。
  只记最后一个窗口；窗口若已关闭或已被你手动还原，则按「手势无动作」处理，不会误操作别的窗口。
  在普通窗口上这几个手势行为不变。

## [1.0.13] - 2026-07-20

### 修复

- **开机自启后托盘图标不出现**（进程在后台正常运行、手势可用，但托盘没有图标，于是无法重载
  配置或退出）。成因有两层：开机时资源管理器还没建好托盘区，`NIM_ADD` 失败；而本该兜底的
  `TaskbarCreated` 广播又被 UIPI 拦下——本程序以管理员权限运行，完整性级别高于广播方
  explorer.exe，而 Windows 只允许消息从同级或更高流向更低。现在两处都修：用
  `ChangeWindowMessageFilterEx` 放行该广播，并在 `NIM_ADD` 失败后定时重试补加（2 秒一次、
  最多 3 分钟），不再单靠广播送达。
- 日志中「当前版本」只打出 `1`：窄日志里误传了宽字符串（`L"1.0.13"` 的字节遇到第一个 NUL
  即截断）。现按窄串输出完整版本号。
- **开机自启任务带着三项错误的默认设置**（此前用 `schtasks` 命令行创建，而这些项命令行根本
  设不了）：
  - `ExecutionTimeLimit=PT72H` —— 连续运行满 72 小时会被任务计划**强行终止**，表现为「跑几天
    就莫名消失」；现改为 `PT0S`（不限时）。
  - `DisallowStartIfOnBatteries=true` —— 笔记本**用电池时根本不自启**；现改为 `false`。
  - `StopIfGoingOnBatteries=true` —— 运行中**一拔电源就被停掉**；现改为 `false`。

  改用任务计划程序的 COM 接口创建任务（`RegisterTask` 直接接收 XML 字符串，不落临时文件——
  临时 XML 会成为提权面：它用于创建最高权限任务，被掉包即等于替攻击者装了个高权限任务）。
  顺带登录触发加了 20 秒延迟，避开开机时资源管理器尚未就绪的竞态。
  **老用户无需手动操作**：启动时会自动用新设置重建该任务。

## [1.0.12] - 2026-07-18

### 新增

- **检查更新遇 GitHub API 限流时自动兜底**。匿名 REST API 额度是 60 次/小时/IP，在 CGNAT
  或公司网络下多人共用出口 IP，很容易被打满而报错。现在仍以正规 API 优先，只有真被限流时才
  自动改走 `github.com` 网页端点（不受该限额），检查与下载照常完成；其它网络错误仍如实报错。
- 发布新增 `checksums.txt`：兜底路径拿不到 API 的校验值，改用它做 SHA-256 校验，
  自替换的 exe 不会在无校验的情况下落盘。

### 变更

- Release 正文精简为纯变更内容：不再内嵌下载表格与 markdown 标记，更新弹窗里不再出现
  一堆符号（资产清单由 GitHub 的 Assets 区自动展示）。
- Release 构建开启全程序优化（LTCG）并优先体积，exe 体积减小约 7%。

## [1.0.11] - 2026-07-18

### 变更

- **自更新资产改为无版本号命名**（`hua-<arch>.exe`，如 `hua-x64.exe`）。更新库就地替换运行中的
  exe、保持磁盘文件名不变，故资产名不再带版本号——直接运行资产者不会再看到「文件名停在旧版本」。
  推荐经 zip 安装：解压即得 `hua.exe`，自更新后文件名恒定，claunch 等快速启动工具绑定不失效。
- 过渡期发布仍附带旧命名 `hua-v<ver>-<arch>.exe`，供内置旧 glob（≤1.0.10）的客户端继续更新。

## [1.0.9] - 2026-07-18

首个公开发布版本：一个纯 C（C17）+ Win32 的 Windows 鼠标手势工具，单文件、静态链接
CRT、零第三方运行时依赖。本版同时补齐了三架构发布与内置自动更新。

### 功能

- **鼠标手势识别**：按住触发键（右键 / 中键 / X1 / X2，可配置）划出方向手势，抬起即执行绑定动作。
  八方向九宫格编码（`8=上 2=下 4=左 6=右` 与四个对角），支持多段折线（如 `26` 下右、`39` V 形）。
- **动作类型**：`key:`（模拟按键，如 `key:ctrl+w`）、`cmd:`（内置命令，如最小化 / 关闭窗口 / 滚动到顶/底 /
  最大化切换 / 打开程序目录）、`cmd:none`（显式屏蔽某手势）。
- **容错匹配**：`Tolerance` 控制方向串的最大编辑距离（默认 `0`，精确匹配）。
- **实时浮层**：绘制过程显示轨迹线与命中动作名，浮层尺寸按手势所在屏 DPI 缩放；未命中显示「手势无动作」。
- **程序专属覆盖**：`[App:xxx.exe]` 节可为指定程序覆盖全局手势；黑/白名单与全屏禁用可配。
- **配置热加载**：`hua.ini` 改完保存即时生效（`ReadDirectoryChangesW` 监听，按内容指纹去抖）。
- **无界面 + 系统托盘**：重载配置 / 打开配置 / 开机自启 / 检查更新 / 打开项目页 / 退出。
- **开机自启**：任务计划程序 + 最高权限（管理员窗口上划手势需要）。
- **配置双位置查找**：exe 同目录 `hua.ini`（便携优先）> `%APPDATA%\hua\hua.ini`（不存在则写内置默认）。

### 新增

- **三架构发布**：GitHub Actions 打 `v*` 标签自动构建并发布 **x64 / x86 / ARM64** 三个架构的单文件 exe
  与含默认配置的 zip。ARM64 设备（Surface Pro X 等）可原生运行。
- **内置自动更新**：托盘「检查更新」走 GitHub Release 单文件 exe 模式（下载 → 校验 SHA-256 → 原子替换 →
  重启）；启动时后台静默检查，发现新版仅弹托盘气泡，不打扰。仅链接系统库 `winhttp` + `bcrypt`。
- **持续集成**：每次 push / PR 都在三架构上编译 + 跑单元测试 + 校验「零运行时依赖」与 PE 架构，发版前同样门禁。

### 修复

- **手势识别：V 形归一推广到全部朝向**。此前只硬编码了 ↓↗（编为 `39`）一个朝向，陡角（≥70°）下另三个
  朝向会退化——↓↖ 退成 `28`、↑↘ 与 ↑↙ 双双退成 `82`（两个肉眼不同的手势撞成同串，无法分别绑定）。
  现按几何对称统一归一：↓↗→`39`、↓↖→`17`、↑↘→`93`、↑↙→`71`。
- **手势识别：V/竖直分界不再由 1 像素决定**。「先竖直向下、再对角上行」与真正的 V 之间原以单臂符号硬翻面
  （右漂 1px 就从 `29` 翻成 `39`）；改用 `MinDistance` 死区，横向张开不足阈值仍判为竖直下上。
- **浮层：淡出不再闪回上一手势的动作名**。上一手势 OSD 尚在淡出途中被新一次点击打断时，隐藏的图层会被
  重新显示，闪回旧动作名残影；淡出改为只呈现当前仍显示的图层。
- **配置写回：AutoStart 改写抽为纯文本变换并单测**。分节 / 就地替换 / 插入 / 补节 / 嵌入 NUL 防护移入
  `ini_edit.c`（零 Win32、可单测），钉住历史四个坑：嵌入 NUL 毁配置、临时路径截断别名到原文件、
  `AutoStartFoo` 前缀误匹配、无节感知追加把裸键塞进 `[App:]` 节。
- **钩子探活 / 状态机健壮性**：探活重装、`ST_TENTATIVE` 超时回退（致命回归修复）、重装时清空采样点、
  托盘广播重建、装钩前先跑完阻塞操作、配置原子写回等一系列修复。
- **移除只写不读的死状态**：ini 监听的停止标志早已被停止事件取代，删除该变量并纠正误导注释。
