# 更新日志

本项目遵循[语义化版本](https://semver.org/lang/zh-CN/)，版本号单一来源为 [`src/version.h`](src/version.h)。
发布 tag（如 `v1.0.9`）必须与该文件一致，否则发布流水线会失败。

## [1.0.9] - 2026-07-18

首个公开发布版本：一个纯 C（C17）+ Win32 的 Windows 鼠标手势工具，单文件、静态链接
CRT、零第三方运行时依赖。本版同时补齐了三架构发布与内置自动更新。

### 功能

- **鼠标手势识别**：按住触发键（右键 / 中键 / X1 / X2，可配置）划出方向手势，抬起即执行绑定动作。
  八方向九宫格编码（`8=上 2=下 4=左 6=右` 与四个对角），支持多段折线（如 `26` 下右、`39` V 形）。
- **动作类型**：`key:`（模拟按键，如 `key:ctrl+w`）、`cmd:`（内置命令，如最小化 / 关闭窗口 / 滚动到顶/底 /
  最大化切换 / 打开程序目录）、`cmd:none`（显式屏蔽某手势）。
- **容错匹配**：`Tolerance` 控制方向串的最大编辑距离（默认 `0`，精确匹配）。
- **实时浮层**：绘制过程显示轨迹线与命中动作名，浮层尺寸按手势所在屏 DPI 缩放；未命中显示「手势无动作」。
- **程序专属覆盖**：`[App:xxx.exe]` 节可为指定程序覆盖全局手势；黑/白名单与全屏禁用可配。
- **配置热加载**：`hua.ini` 改完保存即时生效（`ReadDirectoryChangesW` 监听，按内容指纹去抖）。
- **无界面 + 系统托盘**：重载配置 / 打开配置 / 开机自启 / 检查更新 / 打开项目页 / 退出。
- **开机自启**：任务计划程序 + 最高权限（管理员窗口上划手势需要）。
- **配置双位置查找**：exe 同目录 `hua.ini`（便携优先）> `%APPDATA%\hua\hua.ini`（不存在则写内置默认）。

### 新增

- **三架构发布**：GitHub Actions 打 `v*` 标签自动构建并发布 **x64 / x86 / ARM64** 三个架构的单文件 exe
  与含默认配置的 zip。ARM64 设备（Surface Pro X 等）可原生运行。
- **内置自动更新**：托盘「检查更新」走 GitHub Release 单文件 exe 模式（下载 → 校验 SHA-256 → 原子替换 →
  重启）；启动时后台静默检查，发现新版仅弹托盘气泡，不打扰。仅链接系统库 `winhttp` + `bcrypt`。
- **持续集成**：每次 push / PR 都在三架构上编译 + 跑单元测试 + 校验「零运行时依赖」与 PE 架构，发版前同样门禁。

### 修复

- **手势识别：V 形归一推广到全部朝向**。此前只硬编码了 ↓↗（编为 `39`）一个朝向，陡角（≥70°）下另三个
  朝向会退化——↓↖ 退成 `28`、↑↘ 与 ↑↙ 双双退成 `82`（两个肉眼不同的手势撞成同串，无法分别绑定）。
  现按几何对称统一归一：↓↗→`39`、↓↖→`17`、↑↘→`93`、↑↙→`71`。
- **手势识别：V/竖直分界不再由 1 像素决定**。「先竖直向下、再对角上行」与真正的 V 之间原以单臂符号硬翻面
  （右漂 1px 就从 `29` 翻成 `39`）；改用 `MinDistance` 死区，横向张开不足阈值仍判为竖直下上。
- **浮层：淡出不再闪回上一手势的动作名**。上一手势 OSD 尚在淡出途中被新一次点击打断时，隐藏的图层会被
  重新显示，闪回旧动作名残影；淡出改为只呈现当前仍显示的图层。
- **配置写回：AutoStart 改写抽为纯文本变换并单测**。分节 / 就地替换 / 插入 / 补节 / 嵌入 NUL 防护移入
  `ini_edit.c`（零 Win32、可单测），钉住历史四个坑：嵌入 NUL 毁配置、临时路径截断别名到原文件、
  `AutoStartFoo` 前缀误匹配、无节感知追加把裸键塞进 `[App:]` 节。
- **钩子探活 / 状态机健壮性**：探活重装、`ST_TENTATIVE` 超时回退（致命回归修复）、重装时清空采样点、
  托盘广播重建、装钩前先跑完阻塞操作、配置原子写回等一系列修复。
- **移除只写不读的死状态**：ini 监听的停止标志早已被停止事件取代，删除该变量并纠正误导注释。
